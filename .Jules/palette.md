## 2025-05-02 - Custom Dropzone Keyboard Accessibility
**Learning:** Custom file upload areas (dropzones) built with `<div>` elements natively lack keyboard navigation support. Adding `click` handlers is not enough; without `tabindex`, users cannot reach the control via keyboard. Furthermore, native `Enter` or `Space` activation requires a dedicated `keydown` listener since it's not a native `<button>`. Finally, decorative icons inside the dropzone need `aria-hidden="true"` so screen readers don't read them out needlessly, focusing on the parent's `aria-label`.
**Action:** Always add `role="button"`, `tabindex="0"`, a `keydown` listener for `Enter`/`Space`, and a `:focus-visible` outline to non-native interactive upload elements to ensure full keyboard and screen reader accessibility.
