## 2026-05-26 - Replacing pydub with ffmpeg silencedetect
**Learning:** `pydub`'s `silence.detect_silence` loads the entire audio file into memory and calculates RMS amplitude iteratively in pure Python chunks, leading to significant performance bottlenecks (e.g. taking 7.3s for a short test audio file). Using an external process call to `ffmpeg` and its `silencedetect` audio filter processes the media in heavily optimized native C code, reducing the processing time by ~95% (0.2s for the same file) and lowering memory consumption drastically.
**Action:** Always favor native `ffmpeg` capabilities via `subprocess` for core media analysis (like silence detection, extracting metadata, or cutting) in Python pipelines rather than using heavy abstractions like `pydub` or `moviepy` that read the whole media object into memory.
