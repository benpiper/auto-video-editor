## 2024-05-12 - Secondary Endpoint Path Traversal via JSON Parameters
**Vulnerability:** The `/api/jobs` endpoint accepted `filename` and `bg_image` parameters via JSON requests and directly joined them with the upload folder path without sanitization, leading to a path traversal and arbitrary file read vulnerability (e.g., `../../../../etc/passwd`).
**Learning:** Even if the primary file upload endpoint (`/upload`) correctly sanitizes uploaded files using `secure_filename`, secondary endpoints that receive filenames via JSON payloads from the client must independently sanitize those inputs. Attackers can bypass the initial upload sanitization by sending spoofed JSON payloads to the secondary endpoint.
**Prevention:** Always sanitize any user-supplied file names or paths using `werkzeug.utils.secure_filename` before using them in `os.path.join`, regardless of whether the endpoint is handling a direct file upload or a JSON request referencing a file. Additionally, do not allow arbitrary absolute paths to be passed via API requests.
