## 2025-02-28 - Secure File Handling in Secondary API Endpoints
**Vulnerability:** Path traversal and arbitrary file deletion in `/api/jobs`. The endpoint accepted user-supplied `filename` and `bg_image` from a JSON payload and directly concatenated them using `os.path.join` with the server's `UPLOAD_FOLDER`, enabling a path traversal exploit (e.g. `../../../etc/passwd`). Because the job processing logic automatically deleted these files after completion, this created a critical arbitrary file deletion risk. Furthermore, an explicit conditional allowed fully user-controlled absolute paths for `bg_image`.
**Learning:** Even when primary file upload endpoints secure their inputs, secondary workflow endpoints that reference those uploads by name via JSON/form requests can bypass sanitization if not independently secured.
**Prevention:** Always re-sanitize any user-provided filename or path components using `werkzeug.utils.secure_filename` before using them in file system operations, regardless of whether they represent new uploads or references to existing ones. Avoid accepting explicit absolute paths from client payloads.
