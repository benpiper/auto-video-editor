## 2024-05-24 - Path Traversal in Secondary Endpoints
**Vulnerability:** The `/api/jobs` endpoint in the Flask application accepted a user-provided `filename` and `bg_image` via a JSON payload and directly concatenated them with the `UPLOAD_FOLDER` path. It also allowed absolute paths for `bg_image`.
**Learning:** Even if the primary file upload endpoint (`/api/upload`) sanitizes filenames when saving files, secondary endpoints that reference those files by name in subsequent requests (like creating a job from an already uploaded file) must independently sanitize their inputs. Relying solely on the upload endpoint's sanitization leaves the application vulnerable because attackers can bypass the upload process and directly send malicious payloads (e.g., `../../../../etc/passwd`) to the secondary endpoints.
**Prevention:** Always apply `werkzeug.utils.secure_filename` or similar sanitization functions to *any* user-supplied string that will be used to construct a file path, regardless of whether it's an initial upload or a reference to an existing file. Never trust client-provided paths or allow absolute paths unless strictly necessary and tightly controlled.
