## 2024-05-18 - Prevent Path Traversal in API
**Vulnerability:** The API allowed unsanitized user input in `data.get('filename')` and `data.get('bg_image')` when creating a job, leading to arbitrary file read via Path Traversal (e.g., passing `../../../../etc/passwd`). The background image logic explicitly checked for absolute paths and permitted them.
**Learning:** The secondary endpoints that accept filenames directly via JSON payload to trigger further logic (e.g., creating a processing job using a filename from a previously uploaded file) MUST independently sanitize inputs using `secure_filename`. We cannot rely only on the primary file upload endpoint to handle sanitization, as an attacker can spoof the JSON payload.
**Prevention:** Always use `secure_filename()` on ANY filename provided via JSON payload before joining it with a base directory (like `UPLOAD_FOLDER`), and avoid logic that explicitly permits absolute paths for background images unless carefully controlled.
