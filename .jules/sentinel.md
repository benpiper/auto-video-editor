## 2024-05-04 - Path Traversal in API Endpoint
**Vulnerability:** Found a path traversal vulnerability in `web_app/api.py` within the `create_job` endpoint. The endpoint directly joined `data.get('filename')` with the `UPLOAD_FOLDER` without sanitization. Furthermore, `bg_image` permitted absolute paths and arbitrary path combinations, which could be used to read or delete arbitrary files on the system because of the file cleanup routine in `process_video_async`.
**Learning:** Even when the primary upload endpoint sanitizes file names (using `secure_filename`), endpoints that process job execution based on those filenames from secondary request JSONs (`data.get('filename')`) must also sanitize input to ensure attackers do not bypass the upload restrictions with crafted JSON requests. The `os.path.join()` function when combined with user input represents a significant risk if the second argument is an absolute path or contains traversal characters (`../`).
**Prevention:** Always use `werkzeug.utils.secure_filename` to strip unsafe characters from any user-supplied path components (filenames, background image files) when referencing local file paths. Never allow absolute file paths or unmodified path separators in parameters that determine where files are read from or deleted.
